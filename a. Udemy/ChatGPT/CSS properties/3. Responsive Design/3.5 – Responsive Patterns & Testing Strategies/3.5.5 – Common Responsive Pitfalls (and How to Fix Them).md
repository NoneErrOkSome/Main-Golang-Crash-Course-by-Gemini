## ‚ö†Ô∏è 1. Fixed Widths (`width: 600px`) That Break on Small Screens

### ‚ùå The Problem:

```css
.card {
  width: 600px;
}
```

- On mobile (e.g. 360px wide), this **overflows the screen**
    
- Causes horizontal scrolling, cut-off content, layout bugs
    

### ‚úÖ The Fix:

```css
.card {
  width: 100%;
  max-width: 600px;
}
```

- `width: 100%` keeps it fluid
    
- `max-width` maintains layout limits on large screens
    

---

## ‚ö†Ô∏è 2. Improper Use of `vh` (especially on mobile)

### ‚ùå The Problem:

```css
.hero {
  height: 100vh;
}
```

- On iOS Safari and Android Chrome, `100vh` **doesn‚Äôt account for the dynamic browser UI**
    
- Result: layout jumps or overlaps the address bar
    

### ‚úÖ The Fix:

```css
.hero {
  height: 100svh; /* "safe viewport height" */
}
```

- Use `svh` (supported in most modern browsers)
    
- Or fall back to:
    
    ```css
    height: 100%;
    min-height: 100vh;
    ```
    

---

## ‚ö†Ô∏è 3. Overflow from Long Words or Links

### ‚ùå The Problem:

```html
<p>Thisisareallylongunbreakablewordthatbreaksmylayout</p>
```

### ‚úÖ The Fix:

```css
p {
  overflow-wrap: break-word;
  word-break: break-word;
}
```

Optional:

```css
hyphens: auto;
```

‚úÖ Ensures even long URLs or hashtags don‚Äôt ruin mobile layout

---

## ‚ö†Ô∏è 4. No Flex/Gird Wrapping

### ‚ùå The Problem:

```css
.nav {
  display: flex;
}
```

- When items don‚Äôt fit, they **overflow** or get squashed
    

### ‚úÖ The Fix:

```css
.nav {
  display: flex;
  flex-wrap: wrap;
  gap: 1rem;
}
```

- `flex-wrap` allows elements to flow onto new lines
    
- Always combine with `gap` or `margin`
    

---

## ‚ö†Ô∏è 5. Inputs or Buttons Too Small to Tap

### ‚ùå The Problem:

```css
button {
  padding: 4px 8px;
  font-size: 12px;
}
```

- Hard to tap on touch screens
    
- Fails accessibility standards
    

### ‚úÖ The Fix:

```css
button {
  padding: 0.75rem 1.25rem;
  font-size: 1rem;
  min-width: 44px;
  min-height: 44px; /* Apple HIG and WCAG compliant */
}
```

‚úÖ Bigger tap targets improve usability and mobile friendliness

---

## ‚ö†Ô∏è 6. Sticky or Absolute Headers Overlapping Content

### ‚ùå The Problem:

```css
.header {
  position: sticky;
  top: 0;
}
```

- Content underneath may get **covered**, especially on scroll
    

### ‚úÖ The Fix:

```css
main {
  padding-top: var(--header-height); /* e.g., 64px */
}
```

- Add compensating spacing below header
    
- Use `z-index` and shadows to visually separate
    

---

## ‚ö†Ô∏è 7. No `max-width` on Full-Width Containers

### ‚ùå The Problem:

```css
.container {
  width: 100%;
}
```

- On a 2560px monitor, text stretches across the entire screen
    
- Poor readability and ugly line length
    

### ‚úÖ The Fix:

```css
.container {
  width: 100%;
  max-width: 65ch;
  margin-inline: auto;
  padding-inline: 1rem;
}
```

‚úÖ Controls layout width  
‚úÖ Improves text legibility and layout balance

---

## ‚úÖ Summary Table: Common Pitfalls and Fixes

|‚ùå Problem|‚úÖ Fix|
|---|---|
|Fixed widths (`600px`)|Use `max-width` + `width: 100%`|
|`vh` breaking mobile layout|Use `100svh` or fallback to `min-height`|
|Long strings overflow layout|Use `overflow-wrap: break-word`|
|No wrapping in flex or grid|Use `flex-wrap` or `grid-auto-flow: row`|
|Tiny buttons/inputs|Use `min-width/height: 44px` and proper padding|
|Sticky headers overlap content|Add `padding-top` equal to header height|
|Full-width containers on large screens|Use `max-width` and center with `margin-inline: auto`|
