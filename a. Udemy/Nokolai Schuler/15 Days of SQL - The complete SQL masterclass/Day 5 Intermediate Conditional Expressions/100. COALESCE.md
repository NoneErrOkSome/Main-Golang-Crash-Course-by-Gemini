### üß© `COALESCE()` in SQL

The **`COALESCE()`** function returns the **first non-NULL value** from a list of arguments. It‚Äôs commonly used to handle **missing or null values** in SQL queries.

---

## üîπ **Syntax:**

```sql
COALESCE(value1, value2, ..., valueN)
```

- Returns the **first non-NULL** value.
    
- If **all values are NULL**, it returns `NULL`.
    

---

## üìå **Example Table: `employees`**

|id|first_name|middle_name|last_name|
|---|---|---|---|
|1|Alice|NULL|Brown|
|2|Bob|Alan|Smith|
|3|Carol|NULL|Johnson|

---

### ‚úÖ Example 1: Use COALESCE to fill missing middle names

```sql
SELECT 
  first_name,
  COALESCE(middle_name, '-') AS middle,
  last_name
FROM employees;
```

**Result:**

|first_name|middle|last_name|
|---|---|---|
|Alice|-|Brown|
|Bob|Alan|Smith|
|Carol|-|Johnson|

---

### ‚úÖ Example 2: Full name with fallback

```sql
SELECT 
  COALESCE(first_name, last_name, 'Unknown') AS display_name
FROM employees;
```

- If `first_name` is NULL, show `last_name`.
    
- If both are NULL, show `'Unknown'`.
    

---

## üîπ **Why Use COALESCE Instead of `IS NULL`?**

|Feature|`COALESCE()`|`IS NULL` / `CASE`|
|---|---|---|
|Multiple fallback values|‚úÖ Yes|‚ùå Only single value check|
|Inline replacement|‚úÖ Yes|‚úÖ But more verbose|

---

## ‚úÖ Use Cases:

|Use Case|Example|
|---|---|
|Default value for NULL columns|`COALESCE(salary, 0)`|
|Flexible name formatting|`COALESCE(first_name, last_name)`|
|Avoid NULLs in calculated fields|`price * COALESCE(quantity, 1)`|

---

### üß† Tip:

`COALESCE()` is **portable** and works in **all major SQL databases** ‚Äî better than `IFNULL()` or `NVL()` for cross-platform use.

---

**`COALESCE()`** is the **safeguard against NULLs** ‚Äî helping you keep your SQL outputs clean, readable, and logic-safe.
